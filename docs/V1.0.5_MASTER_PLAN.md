# v1.0.5 主版本 — 桥接规划（Master Plan）

> **状态**：**进行中**（2026-05-27）· **附属**：[V1.0.5.x_MASTER_PLAN.md](./V1.0.5.x_MASTER_PLAN.md)  
> **定位**：**1.0.4.x 收官（`v1.0.4.4`）** 与 **v1.1 世代** 之间的 **桥接小版本**  
> **版本策略**：[VERSIONING.md](./VERSIONING.md) · **长远**：[ROADMAP_V1.1.md](./ROADMAP_V1.1.md)  
> **前置稳定**：生产 **`v1.0.4.4`** · [V1.0.4.x_MASTER_PLAN.md](./V1.0.4.x_MASTER_PLAN.md)

---

## 1. 为何要有 1.0.5（而不是直接 v1.1）

| 原因 | 说明 |
|------|------|
| **1.0.4.x 已宣布收官** | [ROADMAP_V1.0.4.x.md](./ROADMAP_V1.0.4.x.md) 明确 **1.0.4.4 后不再开 `1.0.4.5`** |
| **v1.1 体量过大** | 后台队列 + 协作 M1 + 网关 = 架构级，不宜与「补运维债」混在一个 tag |
| **semver 自然递进** | `1.0.4.4` → **`1.0.5`** 表示 **1.0 线的第五次对外 PATCH 叙事**（非第四段 `1.0.4.5`） |
| **竞品叙事** | 维持 **~2.80**，不宣称 v1.1 能力；对外说「稳定版补丁 + 为 v1.1 铺路」 |

**一句话**：**1.0.5** = **运营与质量债收口 + 体验小修 + v1.1 技术预研文档化**，**不做** IDE-5 重做、**不做** Background Agent。
