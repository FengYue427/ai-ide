# v1.0.3 — Vercel 生产环境变量核对表

> **站点**：https://ai-ide-flame.vercel.app  
> **Phase 1**：[V1.0.3_PHASE1_OPS.md](./V1.0.3_PHASE1_OPS.md)  
> **验收**：`npm run ops:verify-p1`

在 Vercel → Project → **Settings → Environment Variables**（Production）逐项勾选：

---

## 必填

| 变量 | 示例 / 说明 | ☐ |
|------|-------------|:-:|
| `DATABASE_URL` | Neon Postgres 连接串 | ☐ |
| `AUTH_SECRET` | `openssl rand -base64 32` | ☐ |
| `APP_URL` | `https://ai-ide-flame.vercel.app`（**无**尾 `/`） | ☐ |

**验收**：`GET $APP_URL/api/health` → `status: ok`, `database: connected`

---

## 支付宝 Path B（生产收款）

| 变量 | ☐ |
|------|:-:|
| `ALIPAY_APP_ID` | ☐ |
| `ALIPAY_PRIVATE_KEY`（PEM） | ☐ |
| `ALIPAY_PUBLIC_KEY`（支付宝公钥） | ☐ |
| `ALIPAY_SANDBOX` | ☐ **必须未设或 false** |

见 [CN_PAYMENT_SETUP.md](./CN_PAYMENT_SETUP.md)

---

## Cron（订阅到期）

| 变量 | ☐ |
|------|:-:|
| `CRON_SECRET` | ☐ Vercel Cron 自动或手动 |
| `BILLING_CRON_SECRET` | ☐ 可选，本地脚本用 |

**验收**：

```powershell
$env:APP_URL="https://ai-ide-flame.vercel.app"
$env:CRON_SECRET="<secret>"
npm run billing:verify-cron
```

## 自定义域（可选）

绑定域名后 **必须** 同步修改：

1. `APP_URL` = `https://your-domain.com`
2. OAuth App redirect URI（GitHub / Google）
3. 欢迎页页脚应显示新 origin

见 [CUSTOM_DOMAIN.md](./CUSTOM_DOMAIN.md)

---

## 禁止出现在 Production

| 变量 | 原因 |
|------|------|
| `ALLOW_DEV_BILLING=true` | 绕过真实支付 |
| `VITE_ALLOW_OFFLINE_AUTH=true` | 本地假登录 |
| `ALIPAY_SANDBOX=true` | 沙箱误上生产 |

`verify-env --production` 会自动检查。
