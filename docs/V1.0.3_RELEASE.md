## 部署后必做（约 15 分钟）

### 1. Sentry（3-1）

1. [sentry.io](https://sentry.io) 创建项目 → 复制 **Browser DSN**
2. Vercel Production：`VITE_SENTRY_DSN=https://...@sentry.io/...`
3. 重新 Deploy
4. 浏览器控制台执行（仅一次测试）：

```js
throw new Error('ai-ide sentry probe v1.0.3')
```

5. Sentry Issues 中确认 `release: ai-ide@1.0.3`、`environment: production`

详见 [OBSERVABILITY.md](./OBSERVABILITY.md)。

### 2. Vercel Cron（3-4）

- 项目环境变量保留 Vercel 自动生成的 **`CRON_SECRET`**
- 若另设 `BILLING_CRON_SECRET` 供本地脚本，**不必与 CRON_SECRET 相同**（handler 接受任一有效 Bearer）
- 验收：

```powershell
$env:APP_URL="https://ai-ide-flame.vercel.app"
$env:CRON_SECRET="<从 Vercel 复制>"
node scripts/verify-cron-expire.mjs
```

- Vercel → **Cron** 页：次日 `0 3 * * *` 日志应为 **200**

### 3. 运营主体（3-2）

若需公司全称/税号，编辑 [OPERATOR_LEGAL.md](./OPERATOR_LEGAL.md) 后同步 `public/legal/payment*.html`。
