---

## 2. 北极星（MVP）

**在 IDE 内对「当前工作区的一次运行」完成最小调试闭环**：

1. 设断点（编辑器 gutter）  
2. 运行至断点停止  
3. 查看 **调用栈 + 局部变量（只读）**  
4. **继续 / 单步**（step over 优先；into/out 可 P1）  

**非目标（本包）**：多进程 attach 列表、条件断点、watch 表达式、内存分析、C++/Python 调试、远程 SSH 调试。

---

## 3. 技术路线（待 F1 拍板 · 推荐顺序）

| 方案 | 适用 | 优点 | 风险 |
|------|------|------|------|
| **A · WebContainer Node inspect** | 浏览器主路径 | 与现有 Run / WebContainer 一致 | inspect 端口 / 代理需验证 |
| **B · 预览 iframe CDP** | `showPreview` 内页面 | 前端调试体验好 | 与 Node 后端调试分裂 |
| **C · Electron child_process** | 桌面 | 真 Node `--inspect-brk` | 与浏览器能力不一致 |

**MVP 拍板建议**：

- **P0**：**方案 A** — WebContainer 内用户项目 `node` 脚本调试（与 `handleRunCode` 同入口）  
- **P1**：预览页 **方案 B** 仅当 A 稳定后可选 F5+  
- **桌面**：方案 C 作为 **1.1.7.x** patch 或 v1.1.8 前奏，**不挡** 浏览器 MVP GA

**协议**：优先评估 **Debug Adapter Protocol (DAP)** 客户端 + 轻量 adapter；若工期紧，可先 **CDP subset** 直连 inspect WebSocket（文档记录技术债）。
