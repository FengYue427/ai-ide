## 2. 下一阶段：v1.1.0.14～0.20（功能 patch）

### v1.1.0.14 — 从报告恢复队列

- **用户故事**：打开一份队列报告，一键把「失败 / 待执行」项重新入队（Plan 或 Spec）。
- **实现要点**：
  - `queueExecutionReportService`：解析报告 Markdown（Status、Failed、Preview 段）
  - `ChatPanel` / `ReportsSection`：按钮「从报告恢复队列」
  - 与现有 `queuedPlanExecutions` / `queuedSpecExecutions` 合并去重
- **验收**：保存报告 → 清空队列 → 从报告恢复 → 顺序执行与 v1.1.0.6 一致

### v1.1.0.15 — Spec 目录（对齐 Plan 目录）

- **用户故事**：在设置里按 Spec 文件夹浏览，而不是扁平任务列表。
- **实现要点**：
  - `specCatalogService`（路径、标题、未完成数、最近执行时间）
  - `SpecCatalogSection` 或增强 `SpecsSection`：搜索、排序、打开 `tasks.md` / `acceptance.md`
- **验收**：2 个以上 `.aide/specs/*/tasks.md` 可筛选、排序、打开

### v1.1.0.16 — 队列完成时自动留档

- **用户故事**：队列自然跑完（Plan+Spec+Send 皆空）时，自动写入 `.aide/reports/` 并可选浏览器通知。
- **实现要点**：
  - `ideStore` 或 Chat 会话结束钩子检测「队列排空」
  - 设置项：「队列完成时自动保存报告」「完成时桌面通知」（默认关）
- **验收**：跑完 3 步队列 → 无手动点击 → `reports/` 出现新文件；开关可关

### v1.1.0.17 — Plan ↔ Spec 溯源

- **用户故事**：Spec 任务旁显示来源计划步骤；计划步骤旁显示已映射的 Spec 任务。
- **实现要点**：
  - 映射时在 task 行尾追加元数据注释或 `.aide/meta/links.json`（优先轻量：Markdown HTML 注释 `<!-- plan:path#line -->`）
  - `planSpecsBridgeService` 写入；Catalog UI 只读展示
- **验收**：映射后双向可见；删除计划不破坏 Spec 正文

### v1.1.0.18 — 计划总览卡片

- **用户故事**：设置「计划总览」一眼看到所有计划的未完成数、最近执行、队列是否忙。
- **实现要点**：
  - `planOverviewService` 聚合 `planCatalog` + 当前 `ideStore` 队列 + 最近报告时间
  - 小面板：总未完成、进行中队列、跳转 PlansSection
- **验收**：3 个 plan 文件时摘要正确；有队列时显示「执行中」

### v1.1.0.19 — 计划模板

- **用户故事**：从模板新建计划（功能开发 / Bugfix / 发布清单）。
- **实现要点**：
  - `.aide/plans/_templates/*.md` 或内置 3 套
  - PlansSection「从模板创建」→ 写入 `.aide/plans/`
- **验收**：选模板 → 新 plan 文件 → 可立刻勾选步骤执行

### v1.1.0.20 — 报告归档与清理

- **用户故事**：批量归档旧报告、按日期删除，避免 `.aide/reports` 膨胀。
- **实现要点**：
  - `ReportsSection`：多选删除、保留最近 N 份
  - 可选 zip 导出选中报告（复用现有 zip 工具）
- **验收**：10 份报告 → 一键「只保留最近 5 份」
