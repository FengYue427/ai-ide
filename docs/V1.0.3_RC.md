## 目标

在 **不扩功能** 前提下，将 1.0.2.x 累积能力以 **v1.0.3 稳定版叙事** 打包为 RC，完成 live 抽测与 `go-live:preflight` 门禁，为 Phase 3 GA 做准备。

## RC 构建

| 项 | 说明 |
|----|------|
| 版本 | `package.json` → **`1.0.3-rc.1`** |
| 欢迎页 | `VITE_GA_LIVE≠true` 时显示 **「1.0.3 RC」**（来自 `VITE_APP_VERSION`） |
| GA 徽章 | Phase 3 设 `VITE_GA_LIVE=true` →「正式版」 |

## 验收命令

```powershell
npm run go-live:preflight      # test:local + 生产 smoke 5/5
npm run ops:verify-p1          # Phase 1 运维封板
npm run rc:live-spotcheck      # 竞品 live 抽测（自动 + 人工清单）
```

部署后 health 应返回 `version: "1.0.3-rc.1"`：

```powershell
curl https://ai-ide-flame.vercel.app/api/health
```

若生产仍为旧版本，可先：

```powershell
npm run rc:live-spotcheck -- --allow-stale-version
```

## 竞品 live 抽测（5 项）

| # | 能力 | 自动探测 | 人工步骤 |
|---|------|----------|----------|
| 1 | **Chat** | `/api/auth/session`、匿名 `free` 配额 | BYOK 发消息，确认流式与用量 |
| 2 | **Agent + hunk** | bundle 含 `showAgentApplyModal` | Agent 写文件 → 块级 Diff → 应用 |
| 3 | **Tab** | bundle 含 `inlineCompletion` | 设置开 Tab → 编辑器内触发补全 |
| 4 | **@ 索引** | bundle 含 `chat.indexOk` | 导入项目 → `@` 选文件 → 上下文 |
| 5 | **支付宝** | `health.billing.alipay=true` | 套餐页下单（Path B / 沙箱） |
