## 3. 编辑器 diff（F1）

| 项 | 说明 | 优先级 |
|----|------|:------:|
| Tab 模型 | `EditorTab` 扩展：`kind: 'file' \| 'git-diff'` | P0 |
| Monaco | `DiffEditor` 嵌入 `EditorLayout` · 与 theme 同步 | P0 |
| 入口 | `GitPanel`「Diff」→ `openGitDiffTab`（替代或并存 modal） | P0 |
| 关闭 | diff tab 关闭不影响 workdir 文件 tab | P1 |
| Agent | `DiffViewer` modal 保留（apply hunks） | P1 |

## 4. Stage UX（F2）

| 项 | 说明 |
|----|------|
| 分组 | Staged / Changes 视觉对标 VS Code（已有基础，抛光间距与图标） |
| Staged diff | `gitService.getStagedDiff`（HEAD vs index） |
| 快捷 | 状态栏变更数旁「Stage all」；命令面板 `git.stage` |
| 协作 | Viewer 角色禁用 stage/commit（延续 collab read-only） |

## 5. Log drill-down（F3）

| 项 | 说明 |
|----|------|
| 列表 | 保留 50 条 depth · 相对时间 |
| 点击 commit | 展开 / 侧栏显示 `git show` 文件列表 |
| 文件 diff | 对该 commit 打开 diff tab（parent vs commit） |
| 性能 | 大 repo 仅按需读 blob |
