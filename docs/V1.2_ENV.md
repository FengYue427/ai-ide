# v1.2 环境变量与功能开关

> **开发期默认关闭**，避免半成品暴露。  
> **Kickoff**：[V1.2_KICKOFF.md](./V1.2_KICKOFF.md)

---

## 客户端（Vite）

| 变量 | 默认 | 说明 |
|------|:----:|------|
| `VITE_MULTI_ROOT` | `false` | 多根工作区 UI + store（F1） |
| `VITE_PLUGIN_TRUST_MARKET` | `false` | 安装时校验 manifest 签名 / trustTier（F3） |
| `VITE_VIRTUAL_FILE_TREE` | `false` | 大文件树虚拟列表（F2，可先于 multi-root 单独开） |

代码入口：`src/lib/v12Features.ts`

E2E / 手动（无需重建）：

```js
localStorage.setItem('ai-ide:feature:pluginTrustMarket', '1')
```

---

## 服务端（API · F4 可选）

| 变量 | 说明 |
|------|------|
| `PLUGIN_OFFICIAL_PUBLIC_KEY` | Ed25519 公钥（base64），校验官方目录 |
| `PLUGIN_PUBLISH_ENABLED` | `true` 时开放 `POST /api/plugins/publish` 草案（202 + `reviewId`） |
| `PLUGIN_OFFICIAL_PUBLIC_KEY` | 服务端 Ed25519 公钥（base64）；`GET /api/health` → `plugins.officialKeyConfigured` |

---

## 本地开发示例

```bash
# .env.local
VITE_MULTI_ROOT=true
VITE_PLUGIN_TRUST_MARKET=true
npm run dev:stack
```

## 生产建议（v1.2.0 GA 后）

| 开关 | 建议 |
|------|------|
| `VITE_MULTI_ROOT` | GA 稳定 1 周后 → `true` |
| `VITE_PLUGIN_TRUST_MARKET` | 与官方目录同步开启；第三方需审核通过 |
| 既有 `VITE_AI_GATEWAY` | 保持 `true` + 平台 Key |
