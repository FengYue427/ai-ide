# v1.0.6 主版本规划 — 可见能力跃升

> **状态**：规划 → 进行中（2026-05-27）  
> **前置**：v1.0.5.x 桥接 ✅（`v1.0.5.1` live）  
> **主题**：**做看得见、用得到的功能**（Agent 增强 · 文件管理 · 插件生态）  
> **四级附属**：[ROADMAP_V1.0.6.x.md](./ROADMAP_V1.0.6.x.md)  
> **竞品锚点**：v1.0.5.x 维持 **~2.80** → **v1.0.6 目标 ~2.90**

---

## 1. 定位

上一轮（v1.0.5.x）做的是"补债+桥接"，**用户看不到**。  
v1.0.6 反过来：**每个交付都要在 UI 里可见可感受**，真正拉开与竞品的体验差距。

三大方向：

| 方向 | 现状 | v1.0.6 交付 |
|------|------|-------------|
| **Agent** | 工具调用在 activity 区折叠，不透明 | 独立工具面板 · 新增 file 管理工具 |
| **文件管理** | WorkspacePanel 只能导入/删除整体 | 重命名、移动、新建文件夹、批量删除 |
| **插件生态** | 目录仅 2 个 demo | 新增 4 个实用插件 · 筛选与评分 UI |

---

## 2. 四个功能主题

### F1 — Agent 工具面板（Tool Activity Panel）

- **为何**：Agent 调用 6+ 工具时用户完全不知道发生了什么，竞品差距大
- **交付**：Chat 面板右侧 / 可折叠侧边栏，显示每轮工具调用 (tool · input summary · output preview · 耗时)
- **不做**：后台 Agent、流式 token 级实时（v1.1 才做）

### F2 — 文件管理器增强

- **为何**：WorkspacePanel 的文件树仅支持整体导入/删除，不支持重命名、移动、新建文件夹
- **交付**：右键菜单（重命名 · 移动 · 删除 · 新建文件夹）+ 键盘快捷键 (F2/Delete)
- **不做**：跨工作区复制、剪贴板历史

### F3 — 插件目录扩充 + 筛选 UI

- **为何**：目录仅 2 个 demo，用户无法感知插件生态价值；筛选/标签缺失
- **交付**：新增 4 个实用内置插件（JSON 格式化、Markdown 预览增强、代码行数统计、TODO 扫描）+ 标签筛选 + 评分徽章
- **不做**：插件 marketplace 服务端、付费插件

### F4 — Agent 文件管理工具

- **为何**：Agent 无法重命名/移动/创建目录，与 Cursor/Kiro 差距明显
- **交付**：`move_file` · `delete_file` · `create_dir` 三个新 Agent 工具
- **不做**：批量工具、undo history、跨工作区操作
