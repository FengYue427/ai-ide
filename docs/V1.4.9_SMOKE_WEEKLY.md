# v1.4.9 — 生产 smoke 周更清单

> **频率**：每周一次（建议周一）  
> **生效**：v1.4.9 收官起执行，直至 **v1.5.0 GA**  
> **目标 URL**：`https://ai-ide-flame.vercel.app`  
> **门禁**：5/5 通过才视为本周生产健康

---

## 命令

```bash
npm run smoke:production -- https://ai-ide-flame.vercel.app
npm run smoke:report -- https://ai-ide-flame.vercel.app
```

---

## 检查项

| # | 检查 | 期望 |
|---|------|------|
| 1 | health | `status=ok` · `database=connected` |
| 2 | session | 匿名或测试账号 JSON 200 |
| 3 | workspaces | 未登录 **401** |
| 4 | subscription | 匿名 `plan=free` |
| 5 | index | `/` HTTP 200 |

health 附加字段（记录到周报）：

- `version`（部署后应为 `1.4.9+`）
- `plugins.publishEnabled`
- `billing` 支付宝/Stripe 开关状态

---

## 失败处理

| 现象 | 动作 |
|------|------|
| `database=disconnected` | 检查 Vercel `DATABASE_URL` |
| version 滞后 | 触发 Vercel 生产部署 |
| health 5/5 但 E2E 红 | 查 GitHub Actions `test:e2e` / stack / collab |
| 国内 vercel.app 超时 | 见 [BROWSER_LIMITATIONS.md](./BROWSER_LIMITATIONS.md) |

## 与 v1.5.0 启动的关系

- **v1.4.9 tag 部署后**起算：连续 **2 周** 5/5 → 允许启动 **v1.5.0 F1** 编码
