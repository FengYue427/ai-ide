> **前置**：主版本 **[v1.0.4 GA](./V1.0.4_GA_EXECUTION.md)** ✅

---

## 1. 为何要有 1.0.4.x

| 原因 | 说明 |
|------|------|
| **GA ≠ 体验封死** | MCP/感知已在 **1.0.4** 合入，但生产 smoke、语义引导、索引 UX、发布矩阵仍需小步收口 |
| **与 1.0.4 主版本区分** | **1.0.4** = 对外叙事 + Must 功能；**1.0.4.N** = 第四段补丁，**不搞 IDE-5 重做** |
| **竞品锚点** | GA **~2.80** 为初评；**1.0.4.4** 收官时固化文档与 live 抽测，仍 **< 3.0** |

---

## 2. 与 1.0.4 GA 的分工（重要）

| 代号 | 原规划主题 | **1.0.4 GA 已交付** | **1.0.4.x 剩余** |
|------|------------|---------------------|------------------|
| **E1 MCP** | 官方目录 | 预置 ≥3、测试按钮、[MCP_OFFICIAL_CATALOG.md](./MCP_OFFICIAL_CATALOG.md) | 生产 smoke 脚本、FAQ、竞品表 MCP 行终验 |
| **E2 感知** | Agent 上下文 | 焦点文件、终端注入开关、活动行+hunk | 文档边界、手测用例、可选活动摘要持久化 |
| **E3 检索** | @ 与语义 | 基础索引/重试/语义开关 **已有** | **引导 UX**、设置页 onboarding、i18n 补全 |
| **E4 收官** | 发布与评分 | 竞品 §0.1、CSDN 升版 | milestone、live 5 项、Release、**2.80 入档** |

> 因此：**1.0.4.1 / 1.0.4.2 以「深化验收」为主**；**主力开发量在 1.0.4.3 / 1.0.4.4**。

## 3. 四级总览

| 版本 | 代号 | 状态 | 主题 | Git tag 建议 |
|:----:|------|:----:|------|--------------|
| **1.0.4** | — | ✅ GA | 体验巩固 | `v1.0.4` |
| **1.0.4.1** | **E1′ MCP 深化** | ⏳ | 生产 smoke + 文档 + proxy 边界 | 可与 .2 合并 |
| **1.0.4.2** | **E2′ 感知深化** | ⏳ | 手测 + 非 Cascade 话术终稿 | **`v1.0.4.2`** |
| **1.0.4.3** | **E3 检索** | ⏳ | 语义/@ 引导、索引 UX、i18n | 可与 .4 合并 |
| **1.0.4.4** | **E4 收官** | ⏳ | 发布矩阵、milestone、live | **`v1.0.4.4`** |

**合并部署**（推荐，省 Vercel 次数）：

- `1.0.4.1` + `1.0.4.2` → 一次 deploy → tag **`v1.0.4.2`**
- `1.0.4.3` + `1.0.4.4` → 一次 deploy → tag **`v1.0.4.4`**

**闪击路径**（若人力紧）：跳过单独发 `1.0.4.1`，直接 **`1.0.4.3` → `1.0.4.4`**。

## 8. 发包习惯

每附属：

1. `package.json` → `1.0.4.N`
2. `CHANGELOG.md` → `[1.0.4.N]`
3. `npm run test:local` + `go-live:preflight` + `rc:live-spotcheck`
4. `git tag v1.0.4.N`（或合并 tag 见 §3）
5. Web deploy；桌面随 tag 发 Release（无差分可同 tag）
