## 2. v1.2.0 研发阶段（F1～F6）

### F1 — 多根工作区

| 项 | 说明 | 优先级 |
|----|------|:------:|
| 模型 | `WorkspaceRoot { id, name, files, autosaveKey }` · `ideStore.roots[]` + `activeRootId` | P0 |
| UI | ActivityBar 或 FileSidebar 根切换器 | P0 |
| 持久化 | IndexedDB / 云同步按 `rootId` 分桶 | P0 |
| 协作 | 房间绑定 **主根**；次根只读或禁止加入（文档约定） | P1 |
| ADR | [ADR_V1.2_MULTI_ROOT.md](./ADR_V1.2_MULTI_ROOT.md) | P0 |

**验收**：A 根 `index.js` 与 B 根同名文件互不覆盖；切换根后 tab 列表重置或按根缓存。

### F2 — 大文件树

| 项 | 说明 | 优先级 |
|----|------|:------:|
| 虚拟列表 | FileSidebar ≥500 节点可滚动 | P0 |
| 默认折叠 | ≥250 文件时 depth-1 折叠 + hint | P1 |
| 索引 | `projectIndexManager` 按根隔离 | P0 |
| 云端 | 沿用 [WORKSPACE_F1_AUDIT.md](./WORKSPACE_F1_AUDIT.md) 裁剪规则 | P1 |

### F3 — 插件信任（客户端）

| 项 | 说明 | 优先级 |
|----|------|:------:|
| Manifest | 可选 `publisher` + `signature`（Ed25519） | P0 |
| 安装门控 | `installCatalogEntry` 校验签名或 `trustTier: official` | P0 |
| UI | PluginManager 显示信任徽章 / 未签名警告 | P1 |
| 沙箱 | 与 `pluginSandboxRunner` 权限表对齐 | P1 |

详见 [ADR_V1.2_PLUGIN_TRUST.md](./ADR_V1.2_PLUGIN_TRUST.md)。

### F4 — 发布与审核（服务端草案）

| 项 | 说明 | 优先级 |
|----|------|:------:|
| API | `POST /api/plugins/publish`（stub 202 + 人工审核队列） | P2 |
| 目录 | `PLUGIN_CATALOG` 增加 `trustTier` · `signature` 字段 | P0 |
| 官方密钥 | `PLUGIN_OFFICIAL_PUBLIC_KEY` env | P0 |

### F5 — 测试与文档

| 项 | 说明 |
|----|------|
| 单元 | `v12Features` · 签名 verify · store 多根 reducer |
| E2E | 切换根 + 安装官方插件 smoke |
| 文档 | `PLUGIN_SDK_V2.md` 增加「发布与签名」章 |

### F6 — GA

| 项 | 说明 |
|----|------|
| Tag | `v1.2.0` |
| Notes | `RELEASE_NOTES_v1.2.0.md` |
| Deploy | `npm run deploy:check` + smoke 5/5 |
