## 4. 竞品与「真正上架」差距

见 [COMPETITIVE_BENCHMARK_2026.md](./COMPETITIVE_BENCHMARK_2026.md)：

- **综合 ~1.8 vs Cursor ~3.6**：约一代差距，**不是同一档位「主力 IDE」竞品**。
- **优势**：开源、浏览器零安装、BYOK、国内支付骨架。
- **上架含义**：若指 **SaaS 账号产品上架** → 需 D2，当前 **未达标**。
- 若指 **开源 RC 发布公告 + 演示站** → **可以**，但需写明「云账号依赖部署配置」。

## 5. 最终判定

```
┌─────────────────────────────────────────────────────────┐
│  能否「真正上架」？                                        │
├─────────────────────────────────────────────────────────┤
│  RC / 公测公告（BYOK 为主）     →  可以（注明限制）       │
│  D2 MLP（稳定 SaaS 账号产品）   →  不可以（约 3～4 周）   │
│  D3 可收款 GA                   →  不可以（约 2～3 月）   │
│  与 Cursor 正面「全功能 IDE」竞争 →  不适合当前定位       │
└─────────────────────────────────────────────────────────┘
```

**阻塞生产的单一最大因素**：Vercel 环境 **DATABASE_URL 未生效或不可达**（与本地 Neon 偶发断连是不同层问题，但症状一致）。
