## 正文（可直接发）

这条主要做一次阶段复盘，不做过度宣传。

最近把自己的 AI IDE 从 `v1.0.7.2` 连续迭代到了 `v1.1.0.11`。  
核心不是“加了多少功能”，而是把“计划->执行->回填->追踪”这条链路尽量做稳。

### 这轮主要做了什么（按用户感知）

1) **计划系统产品化**  
- 从只能生成计划，变成可管理多个计划（搜索/排序/选择步骤执行）

2) **执行链路补全**  
- 支持计划步骤批量入队  
- 支持映射到 Spec tasks 后继续执行  
- 执行后自动回填结果，并自动勾选完成步骤

3) **失败可控**  
- 失败后可以重试当前步，或者跳过继续  
- 队列防重复入队，避免重复执行

4) **刷新不丢任务**  
- Plan 队列和 Spec 队列都做了本地持久化恢复

5) **可观测性**  
- 当前执行项、队列预览、成功/失败统计、最近完成  
- 支持一键导出会话执行报告（复制到剪贴板）

## 目前仓库里“有什么”（高层梳理）

我也顺便把当前文件结构做了归类，方便自己后续维护：

- `src/components/`：主要 UI 面板（Chat/Settings/Plans/Specs 等）
- `src/services/`：核心业务逻辑（Agent、Plan、Spec、MCP、Queue、Persistence）
- `src/store/`：全局状态（Zustand）
- `src/app/`：应用壳层与面板编排
- `api/` + `lib/api/`：Serverless API 与后端处理
- `docs/`：路线图、执行记录、发布与运维文档
- `docs/publish/`：对外发布文案与封面素材
- `scripts/`：发布、校验、联调、运维脚本
- `e2e/` + `*.test.ts`：端到端与单元测试

代码量确实越来越大，所以我现在把重点放在：
- 可回归（测试通过）
- 可追踪（日志/回填/报告）
- 可维护（职责拆分）

---

## 这轮我最满意的一点

不是某个“炫技功能”，而是执行链路终于形成闭环：  
**Plan -> Queue -> Execute -> Backfill -> Observe -> Export**

这能让我更确定每一步到底有没有做完，而不是只停在“AI 回答了”。
