## 正文（可直接发）

这条是 **v1.1.1 计划系统 GA** 的阶段复盘，不是功能堆砌宣传。

从 `v1.0.7.2` 一路迭代到 **`v1.1.1`**，中间经历了 `v1.1.0.x` 功能 patch 和 `v1.1.1.x` 小步收口。  
核心目标只有一个：**计划不只是 Chat 里一段话，而是能执行、能追踪、能恢复的工作流**。

### 现在能做什么（用户视角）

1) **多计划管理**  
模板创建、搜索排序、步骤勾选执行、复制计划、手动标记完成

2) **Plan → Spec → Chat**  
映射到 Spec tasks、映射并执行、双队列顺序跑、失败重试/跳过

3) **可观测 + 报告**  
队列预览、成功/失败统计、报告保存到 `.aide/reports/`、预览后再恢复队列

4) **溯源与总览**  
Plan↔Spec 双向跳转、计划总览、`.aide` 一键同步到工作区索引

5) **刷新不丢**  
队列持久化、Chat 草稿恢复、损坏数据会提示并清空

### 我刻意没吹的

- 不是 Cursor 替代，是浏览器场景下的计划工作流
- 关页不能后台跑长任务（留给 v1.1.2）
- 国内访问 vercel 可能不稳，本地 dev 更靠谱
