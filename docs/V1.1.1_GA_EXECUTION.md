## 2. 能力摘要（对外 / Release Notes）

### 核心闭环（0.5～0.13 基础 + 0.14～0.20 收尾）

- **Plan**：多计划目录、步骤队列、映射 Spec、映射并执行、失败重试/跳过、队列持久化
- **Spec**：目录管理、执行队列、与 Plan 联动
- **报告**：保存到 `.aide/reports/`、目录管理、从报告恢复队列、完成自动留档
- **溯源**：`.aide/meta/plan-spec-links.json` + UI 展示
- **总览**：计划总览卡片（计划数、队列、最近报告）
- **模板**：内置 3 套 + `_templates/` 自定义
- **清理**：报告多选删除、保留最近 N 份、导出 ZIP

### 1.1.1.x 附属

| 版本 | 内容 |
|------|------|
| 1.1.1.1～3 | 队列 UI、持久化 schema、中英 i18n |
| 1.1.1.4～8 | 手动标记完成、溯源跳转、`.aide` 同步索引、复制计划、报告恢复预览 |
| 1.1.1.9 | GA 文档与版本 **1.1.1** |

### 已知限制（诚实写进 Release）

- 编辑器 `files` 与 **工作区索引** 仍可能不一致 → 用「同步 .aide 到工作区索引」缓解
- 国内访问 `*.vercel.app` 可能超时 → 本地 `npm run dev` 或自定义域名
- **无** 关页后台无人值守（→ v1.1.2 P0）

## 3. 黄金路径验收（约 30 分钟）

1. **模板 → 执行**：设置 → 从模板创建计划 → 勾选 2 步 → 执行 → Chat 完成 → plan 步骤变 `[x]`
2. **映射并执行**：映射到 Spec → 映射并执行 → Spec 队列跑完
3. **报告**：保存报告 → 设置报告目录打开 → **预览恢复** → 确认入队
4. **刷新**：F5 后 Plan/Spec 队列与 Chat 草稿仍在
5. **溯源**：计划步骤「打开 Spec」；Spec 来源按钮回到 plan
6. **同步**：计划总览 → 同步 `.aide` 到工作区索引
