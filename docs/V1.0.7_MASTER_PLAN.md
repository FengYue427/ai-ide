# v1.0.7 主版本规划 — 体验抛光（小优化 · 大收获）

> **状态**：**1.0.7 交付**（2026-05-27）  
> **上一版**：[V1.0.6_MASTER_PLAN.md](./V1.0.6_MASTER_PLAN.md)（1.0.6.4 收官）  
> **下一主版本**：v1.1 — [ROADMAP_V1.1.md](./ROADMAP_V1.1.md)

## 主题

在 v1.0.6「可见能力跃升」之后，用 **8 项小而准的体验修复** 降低日常摩擦：聊天可控、侧栏可读、上限可感知、错误可行动。

## 八项交付（V1.0.7 更新内容）

| # | 项 | 用户感受 | 实现要点 |
|---|-----|----------|----------|
| 1 | **聊天可停止生成** | 长回复可随时打断 | `AbortController` + 发送钮变暂停 |
| 2 | **Agent 默认开启** | 打开即 Agent，记住上次 | `localStorage` `ai-ide:chat-agent-mode` 默认 `true` |
| 3 | **左侧文件树** | 有层级、语言名不再全大写 | `FileSidebar` 树形目录；去掉 `text-transform: uppercase` |
| 4 | **文件上限可视化** | 知道还能加多少文件 | 侧栏容量条 + 工具栏 `当前/上限`；浏览器 **500** / 桌面 **2000** |
| 5 | **Chat UI v2** | 面板更大气、代码块可读 | 渐变会话卡、消息卡片、``` 代码块渲染、输入区浮层 |
| 6 | **413 友好提示** | 请求过大知道怎么办 | 识别 413 / payload too large，聊天内给出缩减上下文步骤 |
| 7 | **文件树展开/折叠** | 大项目不再一屏扁平 | 「展开 / 折叠」+ 导入 ≥8 文件自动展开第一层 |
| 8 | **工作区上限与索引对齐** | 不再卡在神秘的 100 个 | `workspaceContextService` 与 `indexLimits` 统一为 500/2000 |

## 非目标（留给 v1.1+）

- 完整 Markdown 渲染 / 流式语法高亮
- 服务端动态抬升 413 限额（需 API 与计费策略）
- 竞品评分复评（计划在 v1.1 Kickoff）
