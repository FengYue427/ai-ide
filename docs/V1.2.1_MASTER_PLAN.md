## 0. 决策摘要

| 项 | 拍板 |
|----|------|
| **北极星** | 调试体验从「CDP MVP」升级为 **可扩展会话层**；跳定义有 **单一 host API** |
| **P0** | `DebugSession` 接口 · CDP 实现与现有 `debugSessionService` 对齐 |
| **P1** | 条件断点（CDP `condition` + 本地 `hitCount`） |
| **P1** | Watch 只读（最多 3 条） |
| **P2** | `languageServiceHost.goToDefinition` 统一 TS/JS（复用 symbol 索引） |
| **非目标** | Python LSP 服务器 · 多 debug adapter · 插件 `debug.*` 写操作 |

## F2 — 条件断点 ✅

| 项 | 说明 |
|----|------|
| 模型 | `DebugBreakpoint.condition?` · `hitCount?`（localStorage） |
| CDP | `Debugger.setBreakpointByUrl` 传 `condition` |
| UI | Debug 面板断点行：条件 / 命中次数；编辑后 CDP 重同步 |
| 逻辑 | `debugBreakpointHitCount.ts` — 未达 N 次自动 `Debugger.resume` |

**验收**：`x > 1` 条件断点仅在表达式为真时暂停；`hitCount` 达 N 次后命中。

---

## F3 — Watch（只读）✅

| 项 | 说明 |
|----|------|
| 存储 | `ai-ide:debug-watch` 最多 3 条 |
| 运行时 | 暂停时 `Runtime.evaluate` + `callFrameId` |
| UI | Debug 面板 Watch 区；继续执行后清空结果 |

**验收**：暂停后输入 `foo.bar` 可见求值；继续后结果清空。
