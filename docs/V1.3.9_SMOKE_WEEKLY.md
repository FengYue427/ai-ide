# v1.3.9 — 生产 smoke 周更清单

> **频率**：每周一次（建议周一）  
> **生效**：v1.3.9 收官起执行，直至 v1.4 GA  
> **目标 URL**：`https://ai-ide-flame.vercel.app`  
> **门禁**：5/5 通过才视为本周生产健康

---

## 命令

```bash
npm run smoke:production -- https://ai-ide-flame.vercel.app
npm run smoke:report -- https://ai-ide-flame.vercel.app
```

可选更严：

```bash
APP_URL=https://ai-ide-flame.vercel.app npm run deploy:check
```

---

## 检查项

| # | 检查 | 期望 |
|---|------|------|
| 1 | health | `status=ok` · `database=connected` · `version` 为 `1.3.x` |
| 2 | session | 匿名或测试账号 JSON 200 |
| 3 | workspaces | 未登录 **401** |
| 4 | subscription | 匿名 `plan=free` |
| 5 | index | `/` HTTP 200 |

health 附加字段（记录到周报）：

- `plugins.publishEnabled`
- `plugins.officialKeyConfigured`
- `billing` 支付宝/Stripe 开关状态

---

## 失败处理

| 现象 | 动作 |
|------|------|
| `database=disconnected` | 检查 Vercel `DATABASE_URL`（Neon pooler + sslmode） |
| version 非 1.3.x | 触发 Vercel 生产部署 |
| health 5/5 但 E2E 红 | 查 GitHub Actions `test:e2e` / `test:e2e:stack` |
| 国内 vercel.app 超时 | 见 [BROWSER_LIMITATIONS.md](./BROWSER_LIMITATIONS.md) 国内节 |

## 与 v1.4 启动的关系

- **v1.3.9 tag 部署后**起算：连续 **2 周** 5/5 → 允许启动 **v1.4.0** 编码  
- 周更输出存档至内部周报或 CI artifact
