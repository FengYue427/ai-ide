> **版本**：RFC v0.1（2026-06-05）

## 1. 目标

把 Tab 补全从「**少行 insert + 指标可观测**」（v1.4 F1）推进到 Cursor **Tab++ 入门档**：

| 能力 | v1.4 现状 | **Tab++ 目标** |
|------|-----------|----------------|
| 幽灵文本 | 单 range insert，最多 `maxLines`（默认 5） | **多行 block ghost**，视觉连续 |
| FIM 形态 | `prefix` + `suffix` 两段 | **`prefix` + `middle` + `suffix`**（fill-in-middle） |
| 上下文 | 前缀 2000 / 后缀 800 字符 | **打开文件头 + 最近编辑 + Spec task** |
| 接受 | 整段 Tab 接受 | **词/行部分接受** · 下一建议 |
| 延迟 | P95 目标 **&lt;800ms** | P95 目标 **&lt;400ms**（v1.5 生产 env） |
| 开关 | `VITE_TAB_FIM_PRODUCTION` | 新增 `VITE_TAB_PLUS_PLUS`（v1.5 默认关，spike 在 1.4.3） |

**非目标（v1.5 F1）**：专用微调模型训练 · VS Code 插件 · 全语言 LSP 补全。

## 2. 现状架构（代码锚点）

```
registerInlineCompletion.ts
  └─ provideInlineCompletions → inlineCompletionService.fetchCompletion
       ├─ tabCompletionStrategy: fim | platform | chat | none
       ├─ fimCompletionService (DeepSeek/Ollama/Qwen prefix+suffix)
       ├─ trimCompletionToMaxLines(maxLines from inlineCompletionPrefs)
       └─ inlineCompletionMetrics + tabCompletionLatencyPercentile (P50/P95)

Monaco 返回：单 item，insertText 从 cursor 点插入
```

| 模块 | 路径 | 职责 |
|------|------|------|
| Provider 注册 | `src/editor/registerInlineCompletion.ts` | debounce · context 读取 |
| 编排 | `src/services/inlineCompletionService.ts` | 策略 · 缓存 · 指标 |
| FIM API | `src/services/fimCompletionService.ts` | 三厂商 FIM |
| 偏好 | `src/lib/inlineCompletionPrefs.ts` | maxLines · debounce · enabled |
| 延迟 | `src/lib/tabCompletionLatencyPercentile.ts` | P95 target 800ms |
| 设置 | `src/components/SettingsTabCompletionCard.tsx` | 指标只读卡 |

## 3. Tab++ 目标架构

```mermaid
flowchart TB
  subgraph input [输入]
    M[Monaco model + position]
    C[Tab++ context bus]
  end
  subgraph core [Tab++ Core]
    CTX[contextAssembler]
    STR[tabPlusPlusStrategy]
    FIM[fimMiddleSegment]
    GHOST[ghostLayoutEngine]
  end
  subgraph out [输出]
    IC[InlineCompletionItem[]]
    MET[metrics P50/P95]
  end
  M --> CTX
  C --> CTX
  CTX --> STR
  STR --> FIM
  FIM --> GHOST
  GHOST --> IC
  IC --> MET
```

### 3.1 `contextAssembler`（F2 深化，F1 预留接口）

```typescript
// 拟：src/lib/tabPlusPlusContext.ts
export interface TabPlusPlusContext {
  prefix: string
  suffix: string
  middle?: string          // 选中区域或 hole
  relatedFileSnippets: Array<{ path: string; head: string }>
  activeSpecTaskLine?: string
  recentEditRanges: Array<{ path: string; line: number }>
}
```

| 来源 | 优先级 | v1.5 阶段 |
|------|:------:|-----------|
| 当前文件 prefix/suffix | P0 | F1 |
| `getTabCompletionMaxLines()` 多行 | P0 | F1 |
| 同目录打开文件头 2～3 个 | P1 | F2 |
| AIDE Runtime 只读 `activeSpecTask` | P1 | F2 |
| 最近 30s 编辑行 | P2 | F2 |

### 3.2 FIM fill-in-middle（F1）

当光标位于「空洞」或选中替换区：

```
prefix = 文件开头 → hole.start
middle = hole 内已有文本（可为空）
suffix = hole.end → 文件结尾
```

**API 映射**：

| Provider | 今日 | Tab++ |
|----------|------|-------|
| DeepSeek | `prompt` + `suffix` | 同上（已支持） |
| Ollama | `/api/generate` prefix/suffix | 同上 |
| Qwen | FIM endpoint | 验证 middle 参数 |
| Chat 回退 | 用户 prompt 拼接 | prompt 显式标注 `MIDDLE` 段 |

新增 `fetchFimMiddleCompletion(ctx: TabPlusPlusContext)` 包装现有 `fetchFimCompletion`。

### 3.3 多行 ghost 布局（F1）

Monaco `InlineCompletion` 支持 `insertText` 含 `\n`；Tab++ 需：

1. **Range 跨行**：`endLineNumber` / `endColumn` 覆盖 ghost 占位区域（非零宽）
2. **whitespace 对齐**：补全首行与当前 indent 对齐（`ghostLayoutEngine.normalizeIndent`）
3. **maxLines 提升**：生产默认 **8**（设置上限 20，现 12）

```typescript
// 拟返回结构
{
  insertText: string,       // 多行
  range: monaco.Range,      // 可能跨行覆盖 middle
  completeBracketPairs: false,
  isInlineEdit?: boolean,   // Monaco 1.84+ inline edit 探测
}
```

**v1.4.3 spike 范围**：仅 (1) 多行 `\n` insert + 特性开关 `VITE_TAB_PLUS_PLUS_POC`；不做部分接受。

### 3.4 部分接受（F2）

| 快捷键 | 行为 |
|--------|------|
| Tab | 接受全部 ghost |
| Ctrl+→ | 接受下一词 |
| Ctrl+↓ | 接受下一行 |
| Esc | 丢弃 |

实现：监听 `editor.onDidChangeCursorPosition` + ghost 状态机；或 Monaco `inlineCompletions` 多 item 链式。

### 3.5 延迟与缓存（F1）

| 参数 | v1.4 | Tab++ |
|------|------|-------|
| P95 目标 | 800ms | **400ms** |
| debounce 生产默认 | 300ms | **250ms** |
| 缓存 key | prefix tail 80 + suffix 40 | 加入 `middle` hash + context 指纹 |
| 并发 | 取消旧 seq | 保持 + **prefetch 下一行**（F2 可选） |

## 7. 风险

| 风险 | 缓解 |
|------|------|
| Monaco 跨行 range 与主题冲突 | spike 1.4.3 早验证 |
| 多行 ghost 性能 | debounce 250ms · 缓存 · 取消 in-flight |
| 非 FIM provider 体感差 | 明确「推荐 DeepSeek Coder / Ollama」 |
| 与 AIDE Runtime 上下文循环依赖 | context bus **只读**快照 |
